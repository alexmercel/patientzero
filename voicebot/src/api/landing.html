PatientZero Architecture

Agentic stress testing for healthcare voice support.

PatientZero places realistic patient personas into live Twilio calls, streams the conversation through Gemini Live, tracks scenario coverage in real time, and turns recordings into QA evidence.

Runtime Shape FastAPI app + in-process agents

Twilio handles the phone leg, Gemini speaks as the patient, and deterministic Python agents coordinate state, tools, recordings, and reports.

System Map

How the live call path is wired

The live path is intentionally conservative: Gemini remains the voice model, while deterministic Python services decide what should happen next.

1

Operator Dashboard

Preview persona/scenarios, start calls, hang up, move to the next scenario, favorite recordings, and review evidence.

2

FastAPI Routes

Expose dashboard APIs, Twilio callbacks, WebSocket endpoints, recording downloads, reports, and agent status.

3

Agent Runtime

Runs deterministic agents in-process and mirrors selected events into the dashboard event stream.

4

Twilio Media Stream

Connects the phone call to `/ws/twilio-media`, sends 8 kHz mu-law audio, and receives synthesized audio back.

5

AudioBridge

Converts audio between Twilio and Gemini, detects barge-in, waits through hold/checking turns, and queues hidden scenario nudges.

6

Gemini Live

Uses the official `google-genai` SDK for realtime patient speech, input/output transcripts, voice selection, and session continuity.

Call Lifecycle

What happens when a test call starts

This is the exact control flow from clicking `Run Test Call` to post-call reports.

  1. Plan preview is consumed.

    `CampaignSelectorAgent` selects the cached persona and scenario order so the started call matches the dashboard preview.

  2. Twilio call is created.

    `CallManager` calls the Twilio REST SDK, embeds run metadata in custom parameters, and configures status, stream, and recording callbacks.

  3. Transcript session resets.

    `TranscriptAgent` clears the live chat view for the new call and scopes future transcript events to the active call SID.

  4. Twilio opens the media WebSocket.

    The `/ws/twilio-media` route receives `start`, creates a `SessionState`, attaches the stream SID, and registers the `AudioBridge`.

  5. Gemini Live connects.

    `GeminiLiveClient` receives persona context plus the current scenario only, then streams audio and transcripts through the official SDK.

  6. Scenario orchestration runs during the call.

    `ScenarioOrchestratorAgent` and `TestCampaignManager` detect asks, waits, terminal pass/fail answers, repeated answers, and graceful transitions.

  7. Dashboard updates live.

    `DashboardEventBus` broadcasts transcript messages, active scenario state, call events, recording events, and testing progress over `/ws/dashboard`.

  8. Call finalizes into artifacts.

    When Twilio sends stop/status/recording callbacks, the run is evaluated, recordings are synced, markdown is generated, and deep analysis is scheduled.

Agents

Small agents and their responsibilities

These agents are deterministic Python coordinators, not separate LLMs. They make the live call path easier to reason about and test.

CallSessionAgent

Wraps call creation, active call listing, and hangup behavior through `CallManager`.

TranscriptAgent

Owns live transcript reset, committed/live transcript state, and call-scoped transcript ingestion.

CampaignSelectorAgent

Owns persona/scenario preview, pending-first planning, operator overrides, and reset behavior.

ScenarioOrchestratorAgent

Owns active scenario state, pass/fail terminal detection, and graceful next/change scenario transitions.

RecordingAgent

Wraps recording listing, favorite flags, markdown links, deep-analysis links, and artifact metadata.

DeepAnalysisAgent

Wraps post-call audio analysis using Gemini Flash with fallback behavior.

DashboardAgent

Assembles dashboard snapshots from manager and agent-owned state without owning live call logic.

Software Stack

Libraries and tools by responsibility

PatientZero is a Python-first app with a plain HTML/CSS/JavaScript dashboard to keep local debugging straightforward.

Web server

FastAPI + Uvicorn

`fastapi>=0.115` serves APIs, HTML, assets, Twilio callbacks, and WebSockets. `uvicorn[standard]>=0.30` runs the ASGI server.

Realtime voice

Twilio + google-genai

`twilio>=9.2` creates calls and TwiML streams. `google-genai>=1.20` powers Gemini Live and deep analysis.

Configuration

Pydantic Settings + dotenv

`pydantic>=2.8`, `pydantic-settings>=2.3`, and `python-dotenv>=1.0` load `.env` and validate runtime settings.

Storage

Local files

Scenarios/personas live in YAML. Recordings, JSON sidecars, markdown reports, deep-analysis reports, logs, and testing state live under `voicebot/`.

Networking

httpx + ngrok

`httpx>=0.27` downloads Twilio recordings. `run_with_ngrok.sh` can expose the app and export `PUBLIC_BASE_URL` for Twilio.

Observability

structlog + dashboard bus

`structlog>=24.2` emits structured events that are persisted to logs and streamed into the dashboard.

Audio utilities

Python audio helpers + ffmpeg

Built-in helpers convert mu-law/PCM and resample audio. Optional `ffmpeg` trims the first seconds before deep analysis.

Quality gate

pytest + pytest-asyncio

`pytest>=8.2` and `pytest-asyncio>=0.23` cover agents, routes, audio bridge, campaign logic, recordings, and Gemini client behavior.

Operator Tools

What the dashboard gives you

Live Chat

Persistent transcript for both sides of the call, active scenario display, graceful `Next Scenario`, and manual scenario selection.

Testing

Next-run preview, persona/scenario selection, coverage status, failure counter, active run state, historical runs, and exact evidence snippets.

Recordings

Sequential audio files, favorites, markdown reports, deep-analysis reports, and direct artifact links for findings worth submitting.

Ops And Logs

Public URL, WebSocket stream URL, active calls, recent calls, dashboard events, and packet-level logs when needed.

Artifacts And APIs

What gets produced

Reports are designed for stress-test review: every important result links back to exact transcript evidence and the source recording.

Recording package

`recording-0001.mp3`, `recording-0001.json`, `recording-0001.md`, and `recording-0001.deep-analysis.md`.

Core dashboard APIs

`/api/dashboard/snapshot`, `/api/testing/report`, `/api/testing/next-call`, `/api/recordings`, and `/api/agents`.

Twilio callbacks

`/twilio/status`, `/twilio/stream-status`, `/twilio/recording-status`, and `/ws/twilio-media`.

Scenario control API

`POST /api/testing/active-runs/{call_sid}/scenario-transition` queues smooth next/select scenario nudges.
